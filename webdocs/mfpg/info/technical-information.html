A Brief overview of how MFPG is implemented

The program works by constructing a layered weighted DAG by taking the score and constructing a linked list (layerlist) of layers. Each layer represents one of the notes in the score and they are linked in the same order as the in the score. Each layer in the linked list is then combined with the Notemapper to copy all Physical Note Representations (PNRs) for the note that the layer represents. Once the layers are all populated with PNRs, the linked list is combined with the ActionSet to construct the Transition Matrices between two connected layers. This transition matrix calculates the transition cost from every PNR in the first layer to every PNR in the second layer. This process is repeated for each layer in the layerlist. Once this is completed, the graph is complete (represented as the layerlist). After this, the shortest path through the graph is found using a shortest path algorithm.

The program itself requires three inputs from the user:
  • Score: The music score in the form of an XMLMusic file
  • Notemapper: A CSV file containing the note mappings for the PNRs
  • DSL: The ActionSet defined using the MFPG DSL as a .mfpg file.
