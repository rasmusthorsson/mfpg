MFPG

MFPG (Music Fingering Position Generator) is a tool for automatically determining the sequence of fingering positions for a piece of music for a bowed string instrument. When a sequence of fingering positions is developed for a piece of music, it is almost always done individually by the musician who is going to play it. While there are a few suggested guidelines for this process, it is generally a personalized process where many personal preferences come into play.

The problem itself arises from the fact that for many bowed string instruments there is an overlap across strings. A lot of notes can be played on several different strings. Therefore, when playing a note, the musician must select what string to play that note on. To make matters worse, since the position of the hand when playing a note on a string can vary, along with which finger is used to play said note, further choices need to be made about how to play that note.

MFPG solves this problem by allowing the user to define their preferences for developing a sequence of fingering positions, and then does the work of developing that sequence for them.

Why MFPG?

As the previous section states, MFPG takes the preferences of a user and generates the sequence of fingering positions for them. But why could the user not just do this themselves? The answer can be found when looking at the problem from a more mathematical viewpoint. The issue of developing a sequence of fingering positions can be simplified into the graph problem of finding the shortest path through a layered weighted DAG. While there are a few good solutions to this problem from an algorithmic perspective, such as using Topological Sorting .

However, this is not how anyone who is figuring out a sequence for fingering positions actually calculates the sequence. This is instead generally done using some form of a greedy algorithm, an algorithm that effectively only concerns itself with the near future and does not consider the far reaching consequences further along the music piece. This can either lead to having to redo the process if certain sequences are found to lead to deadends (where the musician reaches a hand position on a string from where they have no acceptable way to reach the next note), or a suboptimal sequence where smaller 'acceptable' concessions have to be made.

MFPG assists in finding the best sequence by instead running a shortest path algorithm on a constructed graph based on the music piece and the preferences specified by the user. This should then find candidates of sequences which fit the users preferences, without those earlier mentioned 'acceptable' concessions. Even if a musician were to learn to use a short path algorithm to solve this problem, it would still take far longer and be far more tedious than letting a computer do it, since the preferences are usually well defined.

MFPG Interfaces

There are two interfaces for using MFPG: The CLI (command-line interface) and the GUI (graphical user interface). The CLI a text-based interface (currently only tested using bash), while the GUI is a graphical interface and probably the preferred interface in most cases.

Using MFPG

For general usage, MFPG should be used using the MFPG DSL (Domain-specific Language) which is intended as a relatively simple way to descibe the preferences of the musician. There is a specific page with a guide on how to write MFPG DSL files. This should also generally be used in conjunction with a CSV Notemapper (comma-separated value Notemapper) file.

The CSV Notemapper file is trickier to construct than the MFPG DSL file. It contains all the information about the physical attributes of all playable notes. Guides and Tutorials

The guides available are:
  • The GUI Guide: a guide on what the different parts of the graphical user interface does.
  • The CLI Guide: a short guide on what the different command-line interface options do.
  • The DSL Guide: a guide/tutorial on how to define MFPG DSL files. MFPG DSL files are important in order to personalize the fingering position generation to fit your preferred way to play your instrument.
  • The Notemap Builder Guide: a more advanced guide on how to write code for generating CSV files for the Notemap builder.
